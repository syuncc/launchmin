# Internationalization (i18n)

## Core Rule
Every user-facing text MUST use `t()` from react-i18next — NEVER hardcode strings in components.

## Framework
- Library: react-i18next
- Default language: `en`
- Fallback language: `en`

## Supported Languages
`en`, `zh-CN`, `zh-TW`, `ja`, `ko`, `es`, `pt-BR`, `de`, `fr`, `ru`, `tr`, `it`, `pl`

## Development Workflow
When developing any page or component, MUST provide translations for **all supported languages** at once.

## Translation Key Conventions
- Use namespace per module: `auth:login.title`, `users:table.header.name`
- Keys use camelCase: `confirmDelete`, NEVER `confirm-delete` or `confirm_delete`
- Keep keys descriptive and hierarchical: `dashboard:stats.totalUsers`, NEVER `dashboard:tu`
